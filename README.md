# raccoon-example

A clean, fully-commented reference robot built with [RaccoonOS / LibSTP](https://github.com/htl-stp-ecer/raccoon-lib).

Use this as your starting point when reading the documentation or setting up a new robot. Every file is written to be readable first ‚Äî no competition pressure, no half-finished experiments, no commented-out blocks.

> üìñ **Full documentation:** [htl-stp-ecer.github.io/documentation](https://htl-stp-ecer.github.io/documentation)

---

## What This Robot Does

ExampleBot is a simple delivery robot:

1. **Navigate** ‚Äî drives to an object using heading control and a line-sensor stop condition  
2. **Collect** ‚Äî picks up the object with an arm and claw, then wall-aligns to reset heading error  
3. **Deliver** ‚Äî follows a line to the drop-off zone, deposits the object, clears the area

The task is intentionally simple so the *code structure* is the focus, not the game logic.

## Concepts Demonstrated

| File | Concepts |
|------|---------|
| `config/hardware.yml` | Sensor/motor/servo declarations, `!include-merge` |
| `config/robot.yml` | Kinematics type, PID gains, axis constraints, physical dimensions |
| `hardware/defs.py` | `Motor`, `IRSensor`, `SensorGroup`, `ServoPreset` with named positions |
| `hardware/robot.py` | `DifferentialKinematics`, `FusedOdometry`, `UnifiedMotionPidConfig`, sensor positions |
| `m00_setup_mission.py` | `SetupMission`, `calibrate()`, `wait_for_button()` |
| `m01_navigate_to_object_mission.py` | `mark_heading_reference()`, `parallel()`, `.until()` with compound stop condition |
| `m02_collect_object_mission.py` | Reusable custom step, `wall_align_backward()`, `mark_heading_reference()` |
| `m03_deliver_object_mission.py` | `strafe_follow_line_single()`, combined stop conditions, parallel arm motion |
| `steps/arm_steps.py` | `seq()` composition pattern, `defer()` for runtime decisions |

## Running It

```bash
# Clone and enter the project
git clone https://github.com/htl-stp-ecer/raccoon-example.git
cd raccoon-example/examplebot

# Deploy and run on a connected Wombat
raccoon run
```

See [Getting Started](https://htl-stp-ecer.github.io/documentation/00-quick-start) if this is your first time.

## Why This Exists

The competition robots ([drumbot](https://gitlab.com/fallgame2025/drumbot), [conebot](https://gitlab.com/fallgame2025/conebot), [packingbot](https://gitlab.com/fallgame2025/packingbot)) are great for seeing what the platform can do at full scale ‚Äî but they accumulated complexity under competition pressure. Reading them cold as a new team is hard.

This repo is the thing those robots would have looked like if written for clarity from the start.
